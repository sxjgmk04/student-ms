# 毕业论文：开题与创新点（提纲）

> 供撰写开题报告、答辩 PPT 时引用；与导师「跳出传统成绩登记、结合大数据与 AI 环境」的意见对齐。

## 一、课题背景与问题

- 传统学籍系统多停留在**信息录入、查询、成绩单打印**，难以主动服务教学管理决策。
- 大数据与人工智能背景下，教务数据具备**聚合分析、趋势预测、个性化服务**的潜力，但需注意**隐私保护与可解释性**。

## 二、本系统的「新意」落点（可写进创新点）

| 方向 | 本系统实现 | 论文中可升华的表述 |
|------|------------|-------------------|
| 数据视角 | 在成绩、学籍、异动等多表基础上做**专业维度聚合**、**风险名单输出** | 从「小数据录入」扩展到「学情大数据视图」 |
| 智能化（轻量可落地） | **规则引擎**：均分阈值 + 不及格门数 → 风险等级与文字建议（可解释） | 区别于黑箱模型，强调**可解释 AI / 教务领域规则** |
| 人性化 | 欢迎页**分时段问候**、风险列表附带**帮扶建议文案** | 人机交互与管理者体验 |
| 扩展性（展望） | 后端分析服务可替换为 **ML 模型 API**、对接**校级数据仓库** | 技术路线与「后续工作」章节 |

## 三、系统功能结构（建议画图）

- **业务层**：学生/成绩/异动/奖惩/毕业审核/审计（经典 CRUD）。
- **分析层（新增）**：学情聚合、风险扫描、摘要与亮点生成 → `GET /api/analytics/insights`（管理员）；学生个人对比与提示 → `GET /api/analytics/my-insight`。
- **展示层**：数据统计大屏 + **学情智能分析**独立页；欢迎页展示**学生个人学情摘要**（人性化）。

## 四、设计方法与技术路线

- **方法**：需求分析 → 数据库设计 → 前后端分离 → 在经典模块上**增量叠加**分析模块，降低毕设风险。
- **技术栈**：Vue 3 + Element Plus + ECharts；Spring Boot + Spring Security + JPA + MySQL。
- **分析实现路径（当前）**：SQL/JPA 聚合 + 内存统计 + 阈值规则。  
- **演进路径（论文展望）**：离线批处理（如每学期同步教务库）→ 特征工程 → 分类/回归模型（如学业预警）→ 可选对接大语言模型做**自然语言学情摘要**（需脱敏与审核）。

## 五、开发环境

- JDK 17+、Maven、MySQL 8、Node.js；浏览器端开发与生产构建分离（Vite）。

## 六、开题报告可写的「预期成果」

- 完成可运行的学籍管理系统（含权限与审计）。
- 完成**学情智能分析**子系统：专业均分对比图 + 学业风险预警名单 + 可解释说明文字。
- 论文中讨论：**隐私脱敏、规则与模型结合、与完全自动化「黑箱」系统的差异**。
