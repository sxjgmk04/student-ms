# 项目收尾与稳定性说明

本页用于论文初稿阶段的工程说明，聚焦“规则口径、状态含义、趋势判定、回归清单”。

## 1. 学情规则参数（可配置）

后端配置位置：`server-spring/src/main/resources/application.yml`

- `app.analytics.rules.high-avg-threshold`：高风险均分阈值（默认 68）
- `app.analytics.rules.medium-avg-threshold`：中风险均分阈值（默认 75）
- `app.analytics.rules.fail-score-line`：挂科分数线（默认 60）
- `app.analytics.rules.high-risk-fail-count`：高风险挂科门数阈值（默认 2）
- `app.analytics.rules.medium-risk-fail-count`：中风险挂科门数阈值（默认 1）

说明：以上参数支持环境变量覆盖，便于不同学院/年级采用不同策略。

## 2. 跟进状态定义

风险学生支持三种治理状态：

- `待跟进`：系统检出后尚未开展人工干预
- `已沟通`：辅导员/管理人员已完成沟通和计划确认
- `已改善`：阶段性复盘后，学业表现出现可观改善

状态与备注通过接口 `PATCH /api/analytics/risk-follow-up/{studentId}` 更新，并写入审计日志。

## 3. 学生趋势判定逻辑

基于学期均分趋势：

- 至少两学期数据才判定趋势
- 末学期均分 - 首学期均分 >= 5：`上升`
- 末学期均分 - 首学期均分 <= -5：`下降`
- 其余：`平稳`
- 学期样本不足时：`样本不足`

趋势结果用于学生欢迎页提示，不替代人工判断。

## 4. 回归检查清单（论文初稿前）

### 管理员路径

1. 登录管理员账号 -> 打开 `学情智能分析`
2. 检查风险列表中“触发原因”“跟进状态”“跟进备注”是否可见
3. 修改任一学生跟进状态与备注，确认提示成功
4. 打开 `干预记录`，确认能看到对应时间轴记录
5. 刷新页面后再次检查，确认状态与备注保持一致

### 学生路径

1. 登录学生账号 -> 打开 `欢迎页`
2. 检查个人学情卡片中的“趋势标签”“学期趋势图”“行动清单”
3. 打开 `成绩管理`，确认仅显示本人数据

### 构建与测试

1. 前端构建：`npm run build`
2. 后端编译：`mvn -DskipTests compile`
3. 后端测试：`mvn test`（包含 `AnalyticsControllerTest`）
